# Padrões de Código

## Geral

- Código em inglês.
- Documentação pode ser em português.
- Commits pequenos e objetivos.
- Não misturar refatoração grande com feature.
- Sem segredo no repositório.

## Python

- Python 3.12+.
- Ruff.
- Black ou formatter do Ruff.
- Mypy ou Pyright.
- Pydantic v2.
- SQLAlchemy 2.
- Alembic.
- pytest.

## Flutter

- Dart stable.
- Lints estritos.
- Widgets pequenos.
- Modelos imutáveis.
- Estados explícitos.

## TypeScript

- Strict mode.
- ESLint.
- Prettier.
- Zod em entradas externas.
- Evitar `any`.

## Git

Branches:

- `main`
- `develop`
- `feature/...`
- `fix/...`

Formato de commit recomendado: Conventional Commits.
