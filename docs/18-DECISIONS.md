# Registro de Decisões

## ADR-001 — Flutter

**Status:** aceito  
**Decisão:** usar Flutter para iOS e Android.  
**Motivo:** uma base de código, boa performance e velocidade de desenvolvimento.

## ADR-002 — FastAPI

**Status:** aceito  
**Decisão:** usar FastAPI no backend.  
**Motivo:** tipagem, velocidade de desenvolvimento, suporte assíncrono e integração futura com modelos de previsão.

## ADR-003 — Monólito modular

**Status:** aceito  
**Decisão:** não iniciar com microserviços.  
**Motivo:** reduzir complexidade e custo operacional no MVP.

## ADR-004 — PostgreSQL como fonte da verdade

**Status:** aceito  
**Decisão:** usar PostgreSQL como banco principal e Redis apenas como infraestrutura efêmera.

## ADR-005 — Monitoramento híbrido

**Status:** aceito  
**Decisão:** combinar fonte automática com correção humana.

## ADR-006 — Previsão por regras no MVP

**Status:** aceito  
**Decisão:** começar com heurísticas auditáveis antes de machine learning.
